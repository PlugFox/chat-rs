## Хранение размера высот чанков в бд для оптимизации отрисовки

Хранить в локальном SQLite базе данных размер высот чанков
Или перед отправкой сообщения мерять его высоты под 360, 390 и 640 пикселей и сохранять эти данные в бд для
оптимизации отрисовки на устройствах с разными разрешениями у других пользователей.

https://github.com/chenglou/pretext

## Улучшение структур

Вместо повсеместных HashMap можно использовать более специализированные структуры данных,
например, для хранения сообщений и их связей.

- List - для хранения сообщений по их id, если чат не слишком большой
- Roaring Bitmap - для хранения лайков, закладок, статусов и подобных реакций
- Bitmap - для хранения флагов, например, прочитано/непрочитано, типа сообщения
